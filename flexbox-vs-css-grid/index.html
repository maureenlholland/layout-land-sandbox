Flexbox vs CSS Grid (but not really - you can use both!)

A warning about visual order and accessibility

You will notice both CSS Grid and Flexbox have the ability to shuffle an item's visual order regardless of source order. This has consequences for keyboard and screenreader users. It is a potential failure of WCAG Level A 1.3.2 Meaningful Sequence.

Adrian Roselli has made a feature request for developer tools to include a DOM order viewer. Tabbing through a site will indicate the focus order of interactive elements but you will need a tool like Roselli's bookmarklet or the Accessibility Insights Tab Stops Checker to check non-interactive elements for now.

CSS Grid

CSS Grid layouts are always thinking in two-dimensions: rows and columns. It can specify how a particular item aligns with its particular row and column. The item goes where it is told to go regardless of source order (*Please note: this is an accessibility concern).

Flexbox

Flexbox is always thinking in one direction (according to the flex-direction property): rows or columns. It can wrap into multiple rows or columns if needed, but is never positioning based on the particular row or column. Order can be reversed, or re-arranged with the order property on the flex-item (*Please note: this is an accessibility concern).
